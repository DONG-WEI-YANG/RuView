    AP Placement Guide

    Optimal ESP32 node positions for 3D pose reconstruction. Spatial diversity is key.

    Top-down view (4m x 4m room)
    Side view (height stagger)
    1
    Surround the subject — distribute nodes around the perimeter, not clustered on one side
    2
    Vary heights (0.5m~2.0m) — same-height nodes can't distinguish standing from sitting
    3
    Keep 2~4m from centre — within the Fresnel zone for maximum body influence on CSI
    4
    Point antennas inward — ESP32 PCB antenna is directional; aim the main lobe at the activity area
    5
    Avoid metal/mirrors — large reflective surfaces create multipath interference that degrades CSI quality

    OTA Firmware Update

    Update ESP32 firmware wirelessly via HTTP OTA. Nodes must be connected to the same network.

    Checking for firmware binaries...

    How OTA Works

    1. Build firmware with OTA partition: idf.py menuconfig → enable OTA
    2. Place built .bin in firmware/esp32-csi-node/build/
    3. ESP32 nodes fetch updates from http://<server-ip>:8000/api/ota/download/<file>
    4. Configure OTA URL in firmware sdkconfig or via serial command

    System Architecture

    1

    CSI Capture

    ESP32 antenna array (3TX x 6RX) captures Channel State Information at 20-100Hz via UDP

    2

    Phase Sanitization

    Remove hardware phase offset, carrier frequency offset, and sampling frequency offset (linear fit removal)

    3

    Modality Translation

    1D Conv encoder maps sanitized CSI amplitude/phase to pose-correlated feature space

    4

    Temporal Attention

    Attention mechanism weights important time steps, pooling 60-frame windows into pose estimates

    5

    Joint Decoder

    FC decoder outputs 24 joints x 3 coordinates (xyz). Parallel vital signs extraction via FFT

    Vital Signs Extraction Pipeline

    A

    CSI Buffer

    30-second rolling buffer of subcarrier amplitudes (mean across subcarriers)

    B

    Bandpass Filter

    SOS Butterworth: 0.1-0.5Hz (breathing), 0.8-2.0Hz (heart), 1-8Hz (motion)

    C

    FFT Analysis

    Peak detection in frequency domain for BPM estimation + SNR confidence

    D

    HRV/Stress

    Inter-beat interval analysis: RMSSD, SDNN from heart signal peaks

    E

    Sleep/Motion

    Breath regularity + motion intensity classify sleep stage and activity level

    Advantages

    Through Walls

    WiFi CSI penetrates solid barriers — no line of sight required (Wi-Mesh: 2.4cm joint error)

    Privacy-Preserving

    No cameras or visual recording — only RF signal amplitude/phase analysis

    Real-Time

    Maps 24 body regions in real-time. Vital signs extracted from same CSI stream

    Low Cost

    ESP32 ($5) + antenna array. Total hardware cost under $30

    Technical Honesty

    Addressing common WiFi DensePose criticisms (UDN/GitHub community feedback)

    Real CSI Code

    Binary frame parser (magic 0xC5110001), I/Q demodulation, async UDP receiver — not stubs or mock hardware

    No Hidden Mocks

    DEMO badge visible when simulating. LIVE mode activates only with real backend WebSocket connection

    Model Weights Required

    Architecture is real PyTorch (Conv1D+Attn). Weights not included — must train on your CSI data

    Vitals Work Independently

    Breathing, heart rate, HRV extraction from real CSI does NOT require pose model. Works with raw CSI input
